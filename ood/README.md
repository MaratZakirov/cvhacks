# Energy-based OOD on MNIST

Небольшой проект для проверки **energy-based out-of-distribution detection** на MNIST.

Идея эксперимента простая:

- классификатор обучается **без цифры `1`**;
- после обучения модель тестируется на:
  - **ID**: цифра `7`,
  - **OOD**: цифра `1`;
- для каждого примера сохраняются **logits** и **свободная энергия Гельмгольца**  
  \[
  F(x) = -T \log \sum_i e^{f_i(x)/T}
  \]
- отдельный скрипт анализирует сохраненные logits и сравнивает разные score-функции для разделения **ID** и **OOD**.

Проект состоит из **двух файлов**:

1. `train_energy_mnist.py` — обучение классификатора и сохранение logits/energy.
2. `analyze_logits.py` — анализ распределений и сравнение score-функций `U-TS` и `U+TS`.

---

## Что проверяется

Проект нужен для практической проверки следующей идеи:

- классическая energy-based метрика:
  \[
  F = U - TS
  \]
- альтернативная эвристика:
  \[
  U + TS
  \]

Цель — понять, что лучше разделяет **ID** и **OOD** на реальных логитах обученной модели.

## Файл 1: обучение и сбор energy

`train_energy_mnist.py`

### Что делает

Скрипт:

1. скачивает MNIST CSV;
2. обучает классификатор на цифрах:
   ```text
[1][2][3][4][5][6][7][8]
   ```
   то есть **без цифры 1**;
3. после обучения собирает:
   - **ID**: примеры цифры `7`,
   - **OOD**: примеры цифры `1`;
4. считает для них:
   - logits,
   - free energy;
5. сохраняет:
   - `results/logits.csv`
   - `results/energy_ood_histogram.png`

### Архитектура

Используется компактная CNN-модель `SimpleDerfNet`:

- depthwise свертки,
- pointwise `Linear`,
- нелинейность `Derf`,
- финальный `Linear` classifier head.

### Оптимизация

Используется гибридный оптимизатор:

- `Muon` для параметров с `ndim == 2`,
- `AdamW` для остальных параметров.

### Запуск

Пример запуска:

```bash
python train_energy_mnist.py \
    --epochs 20 \
    --batch-size 64 \
    --temperature 1.0 \
    --data-dir ./data \
    --out-dir ./results
```

Если нужен mixed precision:

```bash
python train_energy_mnist.py \
    --epochs 20 \
    --batch-size 64 \
    --temperature 1.0 \
    --data-dir ./data \
    --out-dir ./results \
    --bf16
```

### Аргументы

- `--epochs` — число эпох обучения.
- `--batch-size` — размер батча.
- `--temperature` — температура \(T\) для energy score.
- `--data-dir` — папка с CSV MNIST.
- `--out-dir` — папка для результатов.
- `--bf16` — включить autocast.

## Файл 2: анализ logits

`analyze_logits.py`

### Что делает

Скрипт загружает `logits.csv` и считает:

- свободную энергию:
  \[
  F = U - TS
  \]
- внутреннюю энергию:
  \[
  U = \sum_i p_i E_i
  \]
- энтропию:
  \[
  S = -\sum_i p_i \log p_i
  \]
- альтернативный score:
  \[
  U + TS
  \]

После этого скрипт:

1. сравнивает `U-TS` и `U+TS` по:
   - ROC-AUC,
   - PR-AUC,
   - best F1,
   - gap между средними ID/OOD;
2. строит:
   - ROC curves,
   - гистограммы score distributions;
3. делает **pairwise-анализ**:
   - считает пары ID/OOD, где одна метрика ранжирует правильно, а другая нет;
4. печатает наиболее яркие примеры logits, где:
   - `F` прав,
   - а `U+TS` ошибается.

### Запуск

После того как `train_energy_mnist.py` сохранил `results/logits.csv`, запускается анализ:

```bash
python analyze_logits.py
```

Если путь к файлу задан внутри скрипта, убедись, что он указывает на актуальный `results/logits.csv`.

## Интерпретация метрик

### Free energy

Используется canonical energy score из статьи:

\[
F(x) = -T \log \sum_i e^{f_i(x)/T}
\]

Обычно ожидается:

- **ID** \(\rightarrow\) более низкая energy,
- **OOD** \(\rightarrow\) более высокая energy.

### Внутренняя энергия и энтропия

При обозначении:

\[
E_i = -f_i
\]

получаем распределение по классам:

\[
p_i = \frac{e^{-E_i/T}}{\sum_j e^{-E_j/T}}
\]

и далее:

\[
U = \sum_i p_i E_i
\]

\[
S = -\sum_i p_i \log p_i
\]

\[
F = U - TS
\]

### Альтернативный score

Также исследуется эвристика:

\[
U + TS
\]

Интуитивно она должна сильнее штрафовать неуверенные, высокоэнтропийные logits.  
Но на реальных данных может оказаться, что классический `U-TS` лучше разделяет ID и OOD.

## Логика эксперимента

Почему цифра `1` рассматривается как OOD:

- модель **не обучалась** на классе `1`,
- следовательно, `1` является unseen class относительно обучающего распределения.

Почему цифра `7` берется как ID:

- `7` входит в train distribution,
- модель видела этот класс при обучении.

Это дает простой и контролируемый toy-setup для исследования energy-based OOD detection.

## Что смотреть в результатах

После запуска полезно обратить внимание на:

### 1. Mean energy

Если все идет ожидаемо, то:

- средняя energy для ID ниже,
- средняя energy для OOD выше.

### 2. ROC-AUC

Если `U-TS` дает более высокий ROC-AUC, чем `U+TS`, это означает, что canonical free energy лучше ранжирует пары ID/OOD.

### 3. Гистограммы

Гистограммы показывают, насколько сильно перекрываются score distributions для ID и OOD.

### 4. Pairwise analysis

Это наиболее полезная часть анализа:

- можно увидеть конкретные пары логитов,
- где `F` дает правильный порядок,
- а `U+TS` ломает ranking.

Это помогает перейти от абстрактных aggregate-метрик к практической интуиции.

## Идея проекта в одном абзаце

Этот проект показывает, как на простом примере MNIST можно исследовать физически мотивированную интерпретацию logits через **свободную энергию Гельмгольца**. Сначала обучается классификатор без одного из классов, затем unseen digit трактуется как OOD, после чего сравнивается, насколько хорошо разные score-функции на основе \(U\), \(S\) и \(F\) отделяют ID от OOD.
